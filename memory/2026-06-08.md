Since the system's active JDK is 21 (and fallback SDK in IntelliJ is 17), target 25 caused a compile error. Restored the JVM target to `21` in `pom.xml` and verified compilation with `mvn clean compile` which now completes with a `BUILD SUCCESS`.

## Lessons Learned
- Programmatic styling of Word documents via `python-docx` using direct XML manipulation (e.g. `w:shd`, `w:pBdr`, `w:tblBorders`) yields highly premium results resembling professional designer templates, enhancing student submission quality.
- AI assistant settings sync conflict: When moving projects between VS Code and IntelliJ, AI assistants in different editors might automatically bump build parameters (like bumping JVM target to an unsupported future release version). Checking git log/diff is the fastest way to trace and undo these unintended changes.
